1.liunx命令的格式
学习liunx就是学命令，首先你得知道它的大体格式。
大体为：
命令 [选项] [参数/目标]

命令：你要做什么（ls、cd、vi、mkdir、ping）
选项：调整命令行为（-l、-a、-h）
参数 / 目标：命令作用在谁身上（文件、目录、网址、用户名等）

大体也可分为以下三种内容：
1. 只有命令（无选项、无参数）

ls        # 列出当前目录下的文件和文件夹
pwd       # 显示当前所在路径
whoami    # 查看当前登录的用户名
clear     # 清空终端屏幕内容
date      # 显示当前系统日期和时间

2. 命令 + 参数（无选项）

cd /home      # 切换目录到 /home
vi test.txt   # 用 vi 打开 test.txt 文件
cat hello.txt # 查看 hello.txt 文件内容
mkdir test    # 创建名为 test 的目录

3. 命令 + 选项 + 参数（完整格式）

ls -l /home   # 以详细列表形式查看 /home 目录内容
rm -rf test   # 强制删除 test 目录及内部所有内容
cp -r dir1 dir2    # 递归复制 dir1 目录及内容到 dir2  这里多参数

2.全局直接使用和   需要切换到目标资源下./使用 or 完整路径使用 

  2.1情况 1：直接写名字就能用（如 ls、cd、vi、ping）
    因为这些命令所在的目录，已经在 PATH 里。比如：
  /usr/bin/ls
  /usr/bin/vi
  系统一搜就搜到，所以直接输名字就行。


  2.2 非系统应用命令
    你下载的应用、程序，本质上就是一个自定义命令
  它能接什么选项、什么参数，完全是开发者写代码时自己定义的
  系统（Linux）不负责规定这些选项是什么意思，只负责把你输入的内容传给程序

  2.3
   比如你自己编译的、自己下载的程序：
  ./test
  ./hello
  ./myapp
  原因只有 2 个：
  当前目录不在 PATH 里
  系统不会自动去你当前文件夹找程序。
  ./ 的意思是：从当前目录找
  告诉系统：别去 PATH 里搜了，就在我现在这个文件夹里运行。
